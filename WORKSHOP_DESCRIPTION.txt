[h1]JK Metrics Lite[/h1]

JK Metrics Lite is a lightweight metrics tool and progress tracker for Jump King Nexile maps and custom maps.

It automatically detects your current area and screen, then saves first reach times, stay times, PB (furthest reached position), and screen transitions to local TSV files. The generated data can also be displayed in OBS during a run or reviewed afterward.

[hr]

[h2]Output Folder[/h2]

By default, the mod creates a JKMetricsLite folder in the same folder as the mod. It also creates JKMetricsLite.env if it does not already exist.

With the default Steam install location, the output folder is usually:

[code]C:\Program Files (x86)\Steam\steamapps\workshop\content\1061090\3741632692\JKMetricsLite[/code]

If your Steam library is installed somewhere else, the first part of the path will be different.

To change the output folder, edit OUTPUT_DIR in JKMetricsLite.env. Leave it empty to use the default location. Restart the game after editing JKMetricsLite.env.

Existing overlay HTML files are not overwritten. Delete an HTML file and restart the game to regenerate it.

[hr]

[h2]OBS Overlays[/h2]

After launching the game with the mod enabled, open OBS and add a Browser Source. Enable local file mode, then select one of the generated HTML files from the JKMetricsLite output folder.

[list]
[*][b]area_name.html[/b] - Shows area names, first reach times, stay graphs, and stay times.
[*][b]area_no.html[/b] - Shows area numbers instead of area names.
[*][b]area_name_speedrun.html[/b] - A compact speedrun-focused area table using m s ms time format.
[*][b]screen_timeline.html[/b] - Displays screen transitions as a real-time graph.
[/list]

[hr]

[h2]Jump Activity[/h2]

jump_activity.tsv records timestamped total frames, jumps, and falls while you play.

Open jump_activity.html directly in a browser and select jump_activity.tsv to view hourly jump heatmaps and monthly jumps.

[hr]

[h2]Area and PB Logic[/h2]

Areas are detected from the map's location_settings.xml data.

If multiple areas match the same screen, the area with the highest start value takes priority. For example, screen 10 is treated as LOCATION_FALSE_KINGS_KEEP, not LOCATION_COLOSSAL_DRAIN.

[code]<Location>
  <start>6</start>
  <end>10</end>
  <unlock>6</unlock>
  <name>LOCATION_COLOSSAL_DRAIN</name>
</Location>

<Location>
  <start>10</start>
  <end>14</end>
  <unlock>11</unlock>
  <name>LOCATION_FALSE_KINGS_KEEP</name>
</Location>[/code]

Screens that do not belong to any defined area are ignored for PB, first reach times, and stay time totals. For example, screen 131 is not included in either area below.

[code]<Location>
  <start>124</start>
  <end>130</end>
  <unlock>124</unlock>
  <name>LOCATION_HOUSE_OF_NINE_LIVES</name>
</Location>

<Location>
  <start>132</start>
  <end>138</end>
  <unlock>132</unlock>
  <name>LOCATION_THE_PHANTOM_TOWER</name>
</Location>[/code]

Area numbers are assigned by first reach order, not by screen order. This avoids revealing the map's intended area order during blind play, but hidden or optional areas can change the numbering if you enter them early.

PB means the furthest reached position based on the first-reached area order and the first-reached screen order inside that area.

[hr]

[h2]Reset Behavior[/h2]

Metrics reset automatically when you start a new game. If you continue a previous game, the last saved metrics are carried over.

You can also reset metrics manually from the pause menu with Reset Metrics.
